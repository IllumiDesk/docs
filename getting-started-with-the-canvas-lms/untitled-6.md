# Step 5: Release Assignments to Learners

##  Generate and Release Assignments

With your assignment sources prepared your next task is to generate and release the assignment files to your students. To clarify:

* **Generate assignment**: this step transforms the source notebook into the destination notebook, where the source notebook is the notebook with all answers and the destination notebook is the notebook designated for learners.
* **Release assignment**: release assignments sends the notebooks to a centralized location so that students can then fetch them.

To complete the generate step, click on the icon located within the `Generate` column:

![](https://files.readme.io/98b16f7-generate_assignment.png)

If all goes well you should then see an icon within the `Preview` column. This option is useful since it allows you to view the student's version of the Jupyter Notebook before releasing it to them. Notice how the `### BEGIN SOLUTION` and `### END SOLUTION` tags were replaced with `# YOUR CODE HERE`:

![Solutions are replaced with code stubs during the generate step.](https://files.readme.io/2af0c98-Screen_Shot_2020-02-14_at_1.31.56_PM.png)

Solutions are replaced with code stubs during the generate step.

Great! Almost there. Now all that is left to complete the assignment distribution task is to click on the `Release` button:

![Release assignments to your students.](https://files.readme.io/c239438-release_assignment.png)

Release assignments to your students.

Once successfully released, you will notice that the badge in the status column changes from `draft` to `released`. A modal should also notify you that the release process has successfully completed.

![The released assignment badge should appear after a successful release to your students.](https://files.readme.io/1e793fb-relased_assignment_badge.png)

The released assignment badge should appear after a successful release to your students.
